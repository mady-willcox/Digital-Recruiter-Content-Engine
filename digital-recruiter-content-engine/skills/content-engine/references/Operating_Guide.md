# Digital Recruiter Content Studio — Operating Guide

**Audience:** Anyone setting up or using the Content Studio agent for the first time.
**Time to first post batch:** ~45 minutes (10 min onboarding + 20 min interview + 15 min creation review).

---

## What this system is

The Digital Recruiter Content Studio is a Claude-powered workflow that turns your real expertise — interviews, sales calls, voice memos, notes — into LinkedIn content that sounds like *you*, not like AI.

It runs as a **Claude Project** with a system prompt and five knowledge files. Every time you start a session, Claude reads the knowledge files, follows the SOPs phase-by-phase, and produces content that hits a specific Ideal Client Profile (ICP) you lock in once.

The system has four modes after onboarding:

1. **Interview Mode** — Claude interviews you for 15-20 minutes to extract stories. Best content comes from here.
2. **Prep Mode** — Claude gives you 5-6 prompts to think about between sessions. Lighter alternative.
3. **Creation Mode** — Paste a transcript, get 12-16 LinkedIn posts following the Digital Recruiter SOP.
4. **Newsletter Mode** — Turn source material into an 800+ word newsletter plus a LinkedIn promo post.

## What you need to set up (one-time)

### The five knowledge files

Upload all five to your Claude Project's "Project Knowledge":

| File | What it does |
|------|--------------|
| `Onboarding_SOP.md` | Runs once on first session. Locks your role, ICP, and tone. |
| `Interview_Mode_SOP.md` | Runs the conversational interview that extracts your best stories. Also contains Prep Mode. |
| `Content_Creation_SOP.md` | The big one. Turns transcripts into LinkedIn posts through ~43 discrete steps. |
| `Newsletter_Mode_SOP.md` | Turns content into long-form newsletters plus a LinkedIn promo post. |
| `Memory_Log_Template.md` | The format Claude uses to generate the cross-session memory file. |

### The system prompt

The system prompt is the routing layer — it tells Claude when to read which file. Paste it into the Project's "Custom Instructions" field. The full prompt is in the package you received.

### The Memory Log

The Memory Log (`Client_Memory_Log.md`) is generated by Claude at the end of every session as a downloadable artifact. **Download it. Upload it at the start of the next session.** Without it, Claude has no memory of what content was already created and may repeat topics.

## First session — what to expect

### Step 1: Open a new chat in your Claude Project

You don't need to type a special command. Just say "Hi" or "Let's get started." Claude detects this is a first-time session (no Memory Log uploaded) and runs onboarding.

### Step 2: Onboarding (5-10 minutes, ~5 responses)

Onboarding is five phases:

- **Phase 1 — Role.** Claude asks if you're a Business Owner / BD person, a Recruiter / IC, or a mix. This determines who your content targets.
- **Phase 2 — Experience.** 6+ months in the field, or under 6 months. Routes the intake questions.
- **Phase 3 — Auto-Pull + Intake.** Paste your LinkedIn About section, recent experience, and website text. Claude reads all of it and only asks the questions it couldn't answer from what you pasted.
- **Phase 4 — ICP Construction & Lock.** Claude builds the ICP document. You review every field. **You must explicitly type "Approved. Lock." to move forward.**
- **Phase 5 — Mode Selection.** Claude asks which mode you want to run first.

**Tip:** Voice memos are gold for the intake. Record yourself answering Phase 3 questions out loud, transcribe, paste. Captures your real voice better than typing.

### Step 3: Pick your first mode

If this is genuinely your first session, **start with Interview Mode**. The best content comes from a real conversation. If you don't have 20 minutes, do Prep Mode and come back later.

## Daily workflow — once you're set up

### The standard cadence

The system targets **two posts per week** (Mondays and Thursdays at 9 AM is a typical schedule). One 16-post Creation batch covers about 8 weeks of content.

A realistic rhythm:

- **Every 6-8 weeks:** Do an Interview Mode session (~20 min) → Run Creation Mode on the transcript (~30-60 min depending on Standard vs Express) → Get 16 posts back, scheduled into a calendar.
- **Once a month or so:** Do a Newsletter Mode session.
- **Anytime:** Use Prep Mode if you want prompts to think about between interviews.

### The session loop

Every session follows the same shape:

1. **Open the Project, start a new chat, upload your latest Memory Log.**
2. **Tell Claude what you want to do** ("Start an interview" / "Create content from this transcript" / "Create a newsletter").
3. **Claude runs the SOP phase-by-phase.** Each phase ends with an approval gate. You type "Approved. Lock." (or "Approved. Continue.") to move forward, or you give feedback and Claude rewrites.
4. **At the end, Claude generates a new Memory Log artifact.** Download it. That's what you upload next time.

## Mode-by-mode walkthrough

### Interview Mode

**You say:** "Start an interview."

**What happens:** Claude warms up with light conversation, then asks one open-ended question at a time. You answer. Claude probes for specifics ("What size company?" "Walk me through what you actually said") until the story is fully extracted, then bridges to the next thread.

**Your job:** Talk like you would to a friend over coffee. Tell stories. Use specific numbers, names, and situations. Don't try to be polished — Claude *wants* the messy version because that's where the post-worthy material lives.

**Output:** A transcript of the conversation that becomes source material for Creation Mode.

**Budget:** 15-20 minutes for a good interview. Longer if you're generous with stories.

### Prep Mode

**You say:** "Prep me."

**What happens:** Claude reads your Memory Log, identifies what's underrepresented in your existing content, and gives you 5-6 specific prompts to think about.

**Your job:** Take the prompts. Jot rough notes. Record voice memos. Bring it all to your next Interview Mode session as a launching pad.

**Note:** Prep Mode does NOT generate content. It's reflection prep, not output.

### Creation Mode

**You say:** "Create content" and paste your source material (interview transcript, sales call transcript, voice memo transcript, notes, or even a YouTube video transcript of yourself speaking).

**What happens:** Claude runs ~43 discrete steps across 9 phases:

- **Phase 0:** Lock the source as immutable.
- **Phase 1:** Confirm the locked ICP from your Memory Log.
- **Phase 2:** Generate 12-16 candidate topics, pressure-test each, and audit for duplicates against past posts.
- **Phase 3:** Write hooks (one per topic).
- **Phase 4:** Draft full post bodies.
- **Phase 5:** De-AI the drafts (this is the magic — six sub-passes that strip out AI tells).
- **Phase 6:** Strategic positioning audit.
- **Phase 7:** Formatting freeze.
- **Phase 8:** Distribute the 16 posts into a calendar (4 weeks, M/W/F slots).
- **Phase 9:** Generate updated Memory Log.

**Your job:** Approve each phase or push back when something's off. Don't skip ahead. The approval gates exist because the SOP is built on the assumption that you're checking the work.

**Time budget:**
- **Standard Mode:** ~2-3 hours total across the run, ~12 hard approvals. Recommended for first-timers and complex transcripts.
- **Express Mode:** ~30-60 minutes total, ~8 hard approvals. Bundles sub-phases into single responses. Recommended once you're trained on the SOP and your source material is clean.

To activate Express Mode, just say "Run in Express Mode" anytime during the session.

### Newsletter Mode

**You say:** "Create a newsletter."

**What happens:** Claude runs 8 phases that produce two deliverables:

- An 800+ word email newsletter built around one focused topic with a reframe at the close (not a sales pitch).
- A short LinkedIn promo post (100-200 words) that drives newsletter signups.

**Your job:** Approve the topic, the subject line, the body, and the promo post. Same approval-gate rhythm as Creation Mode.

**Source material:** Same options as Creation Mode — interview transcripts, sales calls, voice memos, notes, or expand on a topic from your Memory Log.

## How to give feedback when something is off

Claude will sometimes produce a draft that doesn't land. The system has a built-in protocol (Rule 10) for this:

- **Don't apologize back.** Claude won't grovel; you don't have to either. Just say what's wrong.
- **Be specific.** "The hook is too generic" beats "I don't like it."
- **Quote the exact sentence** if Claude's second attempt also misses. Specific feedback gets specific fixes.
- **You can always run a "compliance check"** by saying "Run a compliance check on Phase X." Claude will go rule by rule and fix any violations in place.

What NOT to do:

- Don't try to skip phases. The SOP fails when phases are skipped. If you're rushed, use Express Mode — but every step still runs.
- Don't accept a draft that doesn't sound like you. The whole point is content that reads as authentic to your voice. If it doesn't, push back.

## The Memory Log — your most important file

Every session ends with Claude generating an updated `Client_Memory_Log.md` artifact. This file contains:

- Your locked Client Profile (role, ICP, tone, value prop, success stories, hot takes)
- Every post ever created, with theme tags and angles (so Claude can avoid duplicates)
- Every newsletter created
- Interview history with voice notes and stories
- Prep prompts delivered
- Flags and notes for future sessions

**The workflow:**

1. End of session → Claude generates the artifact in the right panel.
2. Click the download button on the artifact.
3. Save it somewhere you'll find next session (Project Knowledge, Drive, whatever).
4. Start of next session → Upload it before saying anything else.

If you forget to upload it, Claude will still work, but it won't know what content you've already made and may repeat topics. The system has anti-repetition baked in (Rule 8) but it can only enforce that against a Memory Log it can see.

## Common questions

**"Can I switch modes mid-session?"**
Yes. Say "Switch to Creation Mode" or "Move to Newsletter Mode" anytime.

**"What if my transcript is really thin?"**
Claude will tell you. For Creation Mode, you need enough material for at least 12 topics. For Newsletter Mode, you need enough for one focused 800+ word piece. If the source is too thin, Claude suggests a quick interview to expand it.

**"Can I paste a sales call instead of doing an interview?"**
Yes. Sales call transcripts, candidate conversation transcripts, podcast appearances, YouTube clips of yourself speaking — all valid source material. Anything where you're talking about your work in your own voice.

**"What if I want to focus a whole batch on one theme?"**
Tell Claude. "I want this batch to focus on [theme]." That activates the user override (Rule 8, point 4) and suspends the anti-repetition rules for that batch only.

**"Can I edit the SOPs?"**
Yes. The knowledge files are yours. If you want to change tone preferences, add new banned phrases, or modify a phase, edit the file and re-upload to your Project. Just be careful — the SOPs are tightly interconnected. Test changes on a small batch first.

**"What if Claude makes up a story or stat?"**
That's a bug, not a feature. The SOPs explicitly forbid invention (Phase 0 source lock-in, Phase 4 source discipline, Phase 5.1b Specificity Costume kill). If you spot it, push back: "This isn't from the source. Rewrite without it." Claude will respect that.

**"Do I need to do all 16 posts at once?"**
The Creation Mode SOP is built around 16-post batches because that's roughly 8 weeks of content at the recommended cadence. You can ask for fewer if your source genuinely doesn't support 16. Claude will tell you when the transcript is too thin for the full count.

## Troubleshooting

**Claude tries to skip onboarding.** Say: "Run the onboarding SOP." It should re-read `Onboarding_SOP.md` and start at Phase 1.

**Claude bundles multiple phases into one response.** Say: "Per Rule 1, run one discrete step per response." It should split.

**Claude's content sounds AI-ish.** This usually means the de-AI passes (Phase 5.0 through 5.3) didn't run properly. Say: "Run a compliance check on Phase 5." Claude will go rule by rule and fix violations in place. Especially watch for em dashes, "Here's the thing," and clean satisfying endings.

**Claude generates the Memory Log inline as text instead of as an artifact.** Say: "Per the system prompt, the Memory Log must be generated as a Claude Artifact, not inline." Claude should regenerate as an artifact.

**Claude won't stop until you've approved the ICP.** That's correct. The ICP Lock (Phase 4) is a hard gate. You must type `Approved. Lock.` before any mode unlocks. If you genuinely want to come back to it later, just close the chat — your progress through onboarding is saved in working memory for that conversation only, so you'll need to re-run it next time. (This is why you should finish onboarding in one sitting if possible.)

## Quick-start summary

1. **Set up the Claude Project.** Upload all five knowledge files. Paste the system prompt into custom instructions.
2. **Open a new chat. Say hi.** Claude runs onboarding.
3. **Finish onboarding (~10 min). Approve the ICP.**
4. **Pick a mode.** First-timers should pick Interview Mode.
5. **Run the mode.** Approve each phase as you go.
6. **End of session: download the Memory Log artifact.**
7. **Next session: upload the Memory Log before doing anything else.**

That's the loop. Two posts a week, one batch every 6-8 weeks, content compounds. Keep posting. Keep connecting. The system rewards consistency.
