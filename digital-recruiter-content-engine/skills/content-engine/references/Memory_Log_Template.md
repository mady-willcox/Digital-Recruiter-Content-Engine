# Memory Log Template

**Purpose:** This document defines the required structure for the `Client_Memory_Log.md` artifact that the agent generates at the end of every session. The user downloads this artifact and uploads it at the start of the next session so the agent has full context.

The Memory Log is the single source of truth for: who the user is, what content has been created, what's been discussed, and what's off-limits to repeat.

---

## REQUIRED STRUCTURE

Every Memory Log generated by the agent follows this exact structure. Do not omit sections. Do not reorder. If a section is empty (e.g., no newsletters created yet), include the section header with a note `[None yet — first newsletter to come]`.

# Client Memory Log

**User:** [Full name]
**Company:** [Company name]
**Last updated:** [YYYY-MM-DD]
**Sessions completed:** [N]

## POSTS CREATED TO DATE

For every post the agent has ever created for this user, list:

```
Post [N] — [Title]
Created: [YYYY-MM-DD batch]
Hook: [First 1–2 lines of the post]
Core angle: [One-sentence summary of what the post argues]
Source: [Which interview / transcript / session this post came from]
```

Posts are listed newest-first. Group by batch if helpful. The full post text is NOT stored here — just the metadata that prevents repetition.

If no posts have been created yet, this section reads: `[No posts yet — first batch coming this session]`

## GENERATION RULES FOR THE AGENT

When the agent generates the Memory Log artifact at end of session:

1. **It is a Claude Artifact, not inline text.** Type: `text/markdown`. Title: `Client_Memory_Log.md`. The user downloads it from the artifact panel.

2. **Pull from working memory, not from the conversation transcript.** The Memory Log captures derived state (Client Profile, post metadata, ICP) — not the full conversation.

3. **If the user uploaded a prior Memory Log this session, the new Memory Log is the prior one PLUS this session's additions.** Do not lose prior content. The Memory Log is cumulative.

4. **Sessions Completed** counter increments by 1 every time a Memory Log is generated.

5. **Last Updated** is always today's date.

6. **Posts and newsletters are listed newest-first** within their sections.

7. **The agent does NOT include the full text of posts or newsletters in the Memory Log.** Only metadata. The full text lives in the user's content doc, not here.

8. **Tell the user to download** after the artifact is generated:
   > "Your updated Memory Log is ready. Click the download button on the artifact (right panel) to save it. Upload it at the start of your next session so I remember everything and never repeat content."

9. **After delivering the Memory Log, display the Strategy Reminder** per the system prompt.

---

## NOTES

- The Memory Log is the agent's persistent state between sessions. Without it, every session starts from scratch.
- If the user repeatedly forgets to upload the prior Memory Log, gently remind them at start of session: "I don't see a Memory Log uploaded. Without it, I might repeat topics we've already covered. Want to upload it before we start?"
- The Memory Log is private to the user. Do not reference it in content. Do not paste it into LinkedIn posts.
